# 라이브러리

좋아. **(1) 너가 넣을 예정인 라이브러리** + **(2) 내가 추천한 라이브러리**를 한 번에 정리하고, 그걸 기준으로 **폴더 구조도 “라이브러리 운용 친화형”으로 다시** 잡아줄게.

---

## 1) MingleRoom 프론트 라이브러리 구성 정리

### ✅ 코어(필수급)

- **React + TypeScript + Vite**
- **react-router-dom**: 라우팅
- **axios**: REST 통신
- **zustand**: 전역 상태(세션/전역 UI 등)
- **@mui/material + @emotion/react + @emotion/styled**: UI 베이스(MUI)
- **lucide-react**: 아이콘
- **framer-motion**: 애니메이션

### ✅ 실시간(회의/채팅 서비스에 필수)

- **@stomp/stompjs + sockjs-client**: 채팅/시그널링 WS(STOMP) 연결

### ✅ 추천(생산성/구조 안정화)

- **@tanstack/react-query (+ devtools)**: 서버 상태 캐싱/로딩/에러/리패치 표준화
- **react-hook-form + zod + @hookform/resolvers**: 폼 + 스키마 검증(로그인/룸 생성/초대)
- **toast 라이브러리 (택1)**

  - `react-hot-toast` (가볍고 빠름) **또는** `sonner` (UI 깔끔)

- **react-error-boundary**: 기능 단위 에러 격리(화이트보드/웹RTC 같은 곳에서 특히 유용)
- **react-virtuoso**: 채팅 메시지 가상화(메시지 많은 방 대비)
- **react-dropzone**: 파일 드래그앤드롭(채팅 파일/보드 이미지)
- **webrtc-adapter**: WebRTC 브라우저 호환성 완충

### ✅ 테스트/목업/배포 운영(추천)

- **msw**: API 목업(프론트 단독 개발/테스트)
- **vitest**: 유닛 테스트
- **playwright**: E2E(로그인→룸 입장→채팅/보드 smoke test)
- **@sentry/react**: 에러 수집/모니터링(배포 후 필수급)
- **react-i18next + i18next**: 다국어(나중 대비, 초기에 골격만 잡아도 좋음)

## 2) 설치 예시(한 번에)

```bash
# runtime
npm i react-router-dom axios zustand
@mui/material @emotion/react @emotion/styled
lucide-react framer-motion
@stomp/stompjs sockjs-client
@tanstack/react-query
react-hook-form zod @hookform/resolvers
react-hot-toast react-error-boundary react-virtuoso react-dropzone webrtc-adapter
react-i18next i18next
@sentry/react

# dev
npm i -D @tanstack/react-query-devtools msw vitest @playwright/test
```

> 토스트는 `react-hot-toast` 대신 `sonner`로 바꾸려면 `react-hot-toast`만 빼고 `sonner` 넣으면 됨.
